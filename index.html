Philosophy

Aequi (Latin for "even" or "balanced") represents our commitment to equilibrium in every layer of the operating system.

Clarity

Every component is understandable and well-documented. Code speaks for itself.

Correctness

Deterministic behavior and safety at all levels. Predictable, reliable systems.

Minimalism

Only what is necessary, nothing more. Less than 50 system calls. Readable codebase.

Balance

Equilibrium between simplicity and functionality. Architecture, design, and spirit.

Architecture

AequiOS is structured in layers, from hardware to userspace.

Core Components

Initium (Bootloader)

  • UEFI boot stub with ARM64/x86_64 support
  • Serial console output
  • Framebuffer initialization
  • Memory map collection

Ordo (Kernel)

  • Main kernel entry point
  • Subsystem initialization
  • System call interface

Memoria (Memory Management)

  • Physical memory manager (bitmap allocator)
  • Virtual memory manager (paging)
  • Kernel heap allocator

Vita (Process Management)

  • Process management
  • Thread management
  • Fair-share scheduler
  • Power-aware scheduling (ARM64)

LumenFS (Filesystem)

  • Simple, auditable design
  • 4KB block size
  • Inode-based structure
  • Custom VFS layer

Serenus (Userspace)

  • Aura: Graphics framework
  • Ratio: Minimalist shell
  • System utilities

Getting Started

Booting

First, copy the git or manually download AequiOS's latest image

git clone --branch v1.0.0 https://github.com/aequiOS/aequiOS.git 

# Navigate to AequiOS directory
cd aequios

# Build for ARM64 (default)
make ARCH=aarch64

# Or build for x86_64
make ARCH=x86_64

Running

# Run in QEMU
# OS hasn't been finialized yet.
